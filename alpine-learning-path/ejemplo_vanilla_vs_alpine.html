    🧠 JavaScript (El Trabajo Sucio) Vanilla JS 
    // 1. Referencias al DOM (Muy frágil)
const lista = document.getElementById('v-lista');
const total = document.getElementById('v-total');
let pedidos = [];

function addVanilla() {
    pedidos.push('Pedido #' + pedidos.length);
    
    // 2. Crear elementos a mano (El Infierno)
    const li = document.createElement('li');
    li.innerText = 'Pedido...';
    
    const btn = document.createElement('button');
    btn.innerText = 'Borrar';
    btn.onclick = function() {
        li.remove(); // Borrar HTML
        // ...lógica para actualizar array y contador...
    };
    
    li.appendChild(btn);
    lista.appendChild(li); // Inyectar en pantalla
    total.innerText = pedidos.length; // Actualizar contador
}
    2. Prueba Interactiva (Alpine)

    Comandas:

    📄 HTML Reactivo (Todo en uno) Alpine.js 
    <!-- 1. Definir el Estado (Cerebro) -->
<div x-data="{ pedidos: [] }">

    <!-- 2. El contador lee el Array directamente -->
    <h3>Comandas: <span x-text="pedidos.length"></span></h3>
    
    <!-- 3. Botón: Modifica el Array -->
    <button @click="pedidos.push('Pedido')">+ Añadir</button>

    <ul>
        <!-- 4. El Bucle: Pinta el Array automáticamente -->
        <template x-for="(p, index) in pedidos">
            <li>
                <span x-text="p"></span>
                <!-- Borrar es solo sacar del Array -->
                <button @click="pedidos.splice(index, 1)">Borrar</button>
            </li>
        </template>
    </ul>
</div>

    🧠 El "Clic" Definitivo

    En Vanilla, eres un albañil poniendo ladrillos a mano por cada comanda.

    En Alpine, eres el arquitecto: dibujas el plano una vez (template x-for), cambias el dato (el Array), y Alpine construye la casa por ti. No hay código JS suelto, todo fluye desde el HTML.

    📖 Diccionario: ¿Qué hace Alpine por debajo?

    Para entender la magia, hay que ver los trucos. Esto es exactamente lo que Alpine te ahorra escribir en Vanilla JS.

    Directiva Alpine Equivalente en Vanilla JS ¿Qué significa mentalmente?
    x-data="{ count: 0 }" let state = { count: 0 }; El Cerebro. Crea una caja de memoria (Estado) exclusiva para este bloque HTML.
    x-text="count" el.innerText = state.count; El Altavoz. Mantiene el texto de la etiqueta sincronizado. Si la variable cambia, el texto cambia solo.
    @click="sumar()" el.addEventListener('click', () => sumar()) La Oreja. Escucha un evento (click, input, keyup) y ejecuta código. (El @ es un atajo para x-on:).
    x-show="abierto" el.style.display = abierto ? '' : 'none'; El Interruptor. Oculta o muestra el elemento (usando CSS display: none) según si la variable es true o false.
    x-for="item in lista" lista.forEach(i => { create(li)... append... }) La Fotocopiadora. Clona la plantilla HTML (`<template>`) por cada elemento que haya en el array.
    :class="{'rojo': error}" if(error) el.classList.add('rojo'); else... El Pintor. Añade o quita clases de CSS dinámicamente si se cumple la condición. (El : es atajo de x-bind:).
